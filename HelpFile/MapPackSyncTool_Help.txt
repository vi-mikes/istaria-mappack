MapPack Sync Tool by Cegaiel.

- MapPack 5.0 (newly implemented) now uses ' \resources_override\mappack\resources ' folder.
- MapPack 4.0 (deprecated) previously used ' \resources_override\resources ' folder; Not the best method for mod compatibility.
- This new path is what differentiates 4.0 vs 5.0 going forward.

The reason we are using ' \resources_override\mappack\resources ' instead is so that it is more mod friendly.
It will help to avoid confusion/conflicts when using other mods which likely shares folders that MapPack 4.0 previously used.

This ensures that any files that exist in the ' \resources_override\mappack ' folder should ONLY contain mappack files.
Hence this folder can be safely deleted, in future, without potentially removing mod files that you might've added.

Add/Sync button fully automates adding (install) and/or syncing (updating files) MapPack 5.0 - which will add/remove/sync/delete any files/folders that differ from downloaded manifest.

Remove button will fully delete BOTH MapPack 4.0 or 5.0 (both files/folders) and switch it back to the original vanilla (default game install) installation.


Browse button:
- Select the folder your Istaria game exists. The selected folder MUST contain istaria.exe (to verify you're in the correct path).
- You can also manually type the path, in the text box (to the left of Browse button), too.
- Once you click Add/Sync or Remove buttons then this path is remembered, each subsequent time the program is started, in future.

Add/Sync button:
- Install MapPack 5.0 if it does not exist (see next line).
- Sync MapPack 5.0 files which includes adding or removing files that differs from the manifest.
- Deletes any EMPTY directories in \resoures_override\mappack folder, after the sync.
- This will also verify and set 'string mapPath = "resources_override/mappack/resources/interface/maps"' which is the new 5.0 path.

Cancel Action button:
- Self-explanatory. This will stop all processes during Add/Sync or Remove buttons. Button is only enabled while processes are running.

Remove button:
- Uninstall MapPack. This will remove BOTH MapPack 5.0 as well any older versions such as 4.0 listed in manifest.
- This will also verify and set 'string mapPath = "resources/interface/maps"' which is the original/vanilla path.

This program does check if you can write files in the appropriate folders, when using Add/Sync, Remove or Update buttons when used. If file writing fails it will attempt to restart the program in Administrator mode (you will need to confirm permissions via popup box).
